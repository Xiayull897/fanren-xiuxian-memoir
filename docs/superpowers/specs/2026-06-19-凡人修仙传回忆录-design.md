# 凡人修仙传 · 角色回忆录 — 设计文档

> 日期：2026-06-19  
> 状态：设计阶段

---

## 1. 项目概述

### 1.1 项目定位

一个记录动漫《凡人修仙传》所有角色生平的回忆录/百科网站。以角色为中心，展示其修仙之路的故事、机缘、境界变化、人际关系、经典台词等。

### 1.2 核心体验目标

- **沉浸感**：让粉丝仿佛翻开一部修仙界的古籍，浏览每个角色的一生
- **游戏化**：借鉴 RPG 游戏 UI，角色信息以属性面板、时间线、装备栏等形式呈现
- **可扩展**：从核心角色起步，逐步覆盖所有角色；从静态网站起步，未来可接入后端

### 1.3 目标用户

- 《凡人修仙传》动画/小说粉丝
- 需要快速回顾角色剧情线的观众
- 对修仙世界观设定感兴趣的爱好者

## 2. 视觉设计

### 2.1 风格方向：G+H（游戏化时间线叙事）

融合 **G（时间线叙事风）** 和 **H（游戏化资料库）**：

- 每位角色有一条纵向修仙时间轴，事件沿时间线展开
- 角色信息以 RPG 属性面板呈现（境界、法宝、功法如装备栏）
- 修为进展有类似等级的视觉反馈
- 整体暗色沉浸式基调，营造修炼世界的氛围

### 2.2 配色逻辑

- **基础色调**：深色背景（夜空/虚空），金色点缀（灵力光辉）
- **角色专属色**：每位角色分配一个主题色，用于法宝光晕、UI 高亮、时间线节点  
  示例：韩立 → 青绿色，紫灵 → 紫色，元瑶 → 幽蓝色，南宫婉 → 月白色

### 2.3 境界 → 宇宙尺度映射

| 境界 | 宇宙尺度 | 视觉意象 |
|---|---|---|
| 凡人（地壳→地幔→地核） | 🌍 地球三层 | 从表面到内核的蜕变，熔岩纹理 |
| 练气 | 🪐 行星 | 单一星球，初具规模 |
| 筑基 | ☀️ 恒星系统（太阳系） | 行星绕恒星，自成体系 |
| 结丹 | 🌌 星系（银河系） | 漩涡星系，金丹如银心 |
| 元婴 | 🫧 本星系群 | 星系集群，影响力跨越虚空 |
| 化神及以上 | TBD | 暂定：超星系团 / 可观测宇宙 / … |

- 每个境界在时间线上以对应宇宙尺度的视觉背景渲染
- 角色属性面板中有一个 **"宇宙视野"指示器**，随修为升级变化

### 2.4 核心交互：储物袋

- 首页/角色列表页展示一个悬浮的**古朴储物袋图标**
- 用户点击储物袋 → 袋口打开 → 角色以**法宝形态**飞出，悬浮空中
- 每个法宝带有角色专属颜色光晕
- 点击法宝 → 展开该角色的详细信息页
- 交互由 CSS 动画 + Vanilla JS 实现，无需复杂框架

## 3. 角色数据模型

### 3.1 数据结构

每个角色一个文件夹，内含 `data.json`（结构化数据）和 `story.md`（长文本叙事）：

```
data/characters/hanli/
├── data.json     # 结构化字段（见下表）
└── story.md      # 传记/故事长文
```

### 3.2 data.json 字段

| 字段 | 类型 | 说明 | 示例 |
|---|---|---|---|
| `id` | string | 唯一标识 | `"hanli"` |
| `name` | string | 角色名 | `"韩立"` |
| `alias` | string[] | 别名/称号 | `["韩跑跑", "韩老魔"]` |
| `color` | string | 专属主题色 | `"#2d8a56"` |
| `realm` | object | 当前境界 | `{ "name": "大乘", "level": 10 }` |
| `realmHistory` | array | 境界变化时间线 | 见下方 |
| `fortunes` | array | 机缘列表 | 见下方 |
| `events` | array | 关键事件列表 | 见下方 |
| `voiceActor` | object | 配音演员 | `{ "cn": "xxx", "jp": "xxx" }` |
| `affiliation` | string | 宗门/势力 | `"黄枫谷"` |
| `spiritualRoot` | string | 灵根 | `"四灵根（伪灵根）"` |
| `relationships` | array | 角色关系 | 见下方 |
| `treasures` | array | 法宝/功法 | `[{ "name": "青竹蜂云剑", "rarity": "本命法宝" }]` |
| `quotes` | array | 经典台词 | `[{ "text": "...", "context": "..." }]` |

#### realmHistory 结构

```json
{
  "realm": "筑基",
  "chapter": "第xxx章",
  "episode": "第x集",
  "event": "在xxx突破至筑基",
  "cosmicScale": "恒星系统"
}
```

#### fortunes / events 结构

```json
{
  "name": "获得掌天瓶",
  "type": "机缘",          // 机缘 | 事件
  "description": "...",
  "realm": "凡人",
  "significance": "命运转折点"
}
```

#### relationships 结构

```json
{
  "name": "南宫婉",
  "relation": "道侣",
  "note": "..."
}
```

### 3.3 story.md 格式

Markdown 自由格式，存放角色传记、故事概述、台词详细背景等长文本。模板：

```md
# 韩立

## 生平概述
...

## 性格特点
...

## 经典台词
- "..."（第x集，对xxx所说）
```

## 4. 页面结构

### 4.1 首页：储物袋 + 法宝选择

- 中央悬浮储物袋（CSS 动画缓慢上下浮动）
- 点击后袋口打开，已收录角色以法宝形态飞出，散布在页面上
- 每个法宝一个悬浮卡片：角色头像/符号 + 名字 + 颜色光晕
- 顶部可选筛选（按境界、宗门等）

### 4.2 角色详情页

布局自上而下：

1. **角色属性面板**（顶部横幅）
   - 名字、称号、宗门、灵根、配音演员
   - "宇宙视野"指示器（当前境界对应的宇宙尺度背景）
   - 法宝/功法以图标 + 稀有度边框展示

2. **境界时间线**（主体区域）
   - 纵向时间轴，节点为境界突破事件
   - 每个节点背景为对应宇宙尺度（行星→恒星系→星系→星系群）
   - 境界间的机缘/事件穿插在时间线上

3. **关系网络**（侧栏或底部）
   - 简易关系图或列表
   - 每个关系可点击跳转到对应角色

4. **经典台词**（特色区域）
   - 引用样式展示，附上下文说明

### 4.3 导航

- 面包屑：首页 > 角色名
- 角色法宝选择器（切换角色无需返回首页）
- 响应式设计：桌面端为主，移动端适配

## 5. 技术方案

### 5.1 技术选型

| 层 | 选择 | 理由 |
|---|---|---|
| 页面 | HTML + CSS + Vanilla JS | 无框架依赖，静态托管即可 |
| 数据 | JSON（结构化）+ MD（长文本） | 人可编辑，程序可解析 |
| 构建 | 无需构建 | 纯静态，浏览器直接读取 JSON |
| 部署 | GitHub Pages / Vercel | 免费，支持自定义域名 |
| 版本管理 | Git | 数据也可版本控制 |

### 5.2 数据加载流程

```
用户打开页面 → fetch('data/characters/hanli/data.json')
            → JS 解析 JSON → 渲染 DOM
            → 可选 fetch('story.md') → 渲染传记区域
```

### 5.3 不依赖项

- ❌ 数据库
- ❌ 后端服务
- ❌ 前端框架（React/Vue 等）
- ❌ 构建工具（Webpack/Vite 等）
- ❌ Node.js 运行时（部署不需要）

## 8. 设计决策记录

| 决策 | 结论 | 理由 |
|---|---|---|
| 数据存储格式 | JSON + MD 混用 | JSON 存结构化数据，MD 存长文本 |
| 是否有数据库 | 暂不 | 先做静态网站，后续再升级 |
| 前端框架 | 无 | 纯静态页面足够，不引入框架复杂度 |
| 视觉风格 | G+H 游戏化时间线 | 用户选择，最契合修仙题材 |
| 交互隐喻 | 储物袋 → 法宝 | 用户创意，沉浸感强 |
| 角色范围 | 先核心后扩展 | 迭代开发，快速上线 |
